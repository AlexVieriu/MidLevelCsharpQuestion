Decorator design pattern:
-> adds extra functionality to an existing object, 
without affecting the behavior of other objects from the same class

Decorator:
-> allows adding behavior to an object dynamically, without touching its code
-> allows following the Open-Closed Principle
-> allows following the Single Responsibility Principle
